# Handoff — 2026-04-20

## Issue fixed: Manage Students empty for teacher with no `school_id`

**Teacher:** fidel@hakuhokai.com

### Symptom
- Manage Students page showed "0 students" / "No students found".
- Reports page showed all the teacher's students correctly.
- Other schools' teachers were unaffected.

### Root cause
`src/hooks/useManageStudents.js` had an early-exit when `appState.currentUser.school_id` was null — `fetchStudents` never queried, so the list stayed empty. Reports works because it filters by `teacher_id` only.

### Code fix (already deployed)
`src/hooks/useManageStudents.js`:
- `fetchStudents` — no longer early-exits on missing `school_id`; falls back to a `teacher_id`-only query.
- Fallback poll — same fallback logic.
- Realtime subscription and `unlinkedStudents` computed value already handled missing `school_id` correctly.

User verified:
1. Affected teacher now sees their students.
2. Other teachers (with `school_id`) still see their students AND unlinked students from same school.
